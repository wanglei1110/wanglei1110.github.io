云服务器使用密码登陆及Syncthing文件备份

发表于2023-09-13|更新于2023-09-13|实用工具
|阅读量:

动机

  • 问题的由来
    我申请了一台AWS云服务器，亚马逊在网页端登陆不能进行文件传输，一般使用ssh客户端传输和运行指令。ssh客户端通过xx.pem秘钥链接，但秘钥每隔一段时间会失效，需要重新加载xx.pem，如果秘钥丢了,会出现文件将不能传输，且云上数据没有备份等尴尬情况。

基于密码的登录云服务器

解决方案：
使用 「用户密码」登陆,在sshd_config文件中启用密码登陆

操作方法（在云服务器网页端操作）：

vim /etc/ssh/sshd_config

取消注释并将其修改yes

#PasswordAuthentication no 
PasswordAuthentication yes
PermitRootLogin：yes

关闭文件并重启ssh服务

sudo service ssh restart

设置普通用户密码

passwd ec2-user # 点击回车，输入自己的密码

验证：
在客户端使用用户登陆，注意只能用普通用户登陆，这里用ec2-user。设置如下，每次登陆时填写密码

Syncthing文件定时自动同步

目标：A设备上使用syncthing定时备份文件夹信息到B设备
A设备为国外云服务器/linux系统，B设备为本地/window10系统

Syncthing简介

  • Syncthing是一个开源的同步工具，可在多平台不同局域网的设备之间同步。它使用点对点（P2P）技术，通过加密TLS通信实现设备间同步；
  • 隐私问题：在Syncthing软件架构中，在设备间是对等的，直接通信，而不经过中央服务器，不用担心数据存储在第三方服务器上带来隐私泄露问题。它还使用正向保密技术加强数据之间的安全性。

安装

Windows安装
B设备安装 SyncTrayzor, 下载地址 https://github.com/canton7/SyncTrayzor/releases
点击 SyncTrayzorSetup-x64.exe

Docker安装
A设备安装：先下拉镜像，通过容器方式启动，在浏览器中输入IP:8384 GUI界面，之后操作和B设备相似

docker pull syncthing/syncthing
docker run -p 8384:8384 -p 22000:22000/tcp -p 21027:21027/udp \
    -v /home/ec2-user:/wherever/st-sync \
    --hostname=my-syncthing \
    syncthing/syncthing:latest

「注」：

  • 需要运行容器前先查看安全组是否包含 tcp 8384，22000，udp 21027 端口，其中udp端口可选
  • 主机目录和挂载到容器内部的指定目录，在A设备填写同步路径时使用容器中的路径。如这里A设备的路径为/wherever/st-sync/hexo_data_new

问题及解决：
Q: 传输速度慢
A: 由于A设备是国外服务器，B设备是国内的，两者第一次同步时传输速度慢是正常。
加速同步的方法:察觉到是B设备的网速较慢，开启全局代理后传输加快（由几KB->几MB）;
修改要传输的文件路径，只传输一些经常改动的文件。之后的传输，没有感觉到速度很慢

Q: 同步文件乱码 如 XX.syncthing-enc文件夹，不是预期的传输内容或重复安装SyncTrayzor后启动失败报错 Failed to start Syncthing. XXX
A: AB两个设备的syncthing版本不一致，一般是windows系统版本低可以尝试以下操作： 设置 -> Syncthing (选项卡) -> 高级 -> 在”Syncthing命令行标志”输入框中输入”-allow-newer-config”（不包含引号） -> 点击保存 -> 再次尝试启动Syncthing。
如果能够成功启动，那么问题应该解决了。
